# NL Automation Test Checklist

## 목적
자연어 자동화가 실사이트에서 안정적으로 동작하는지 반복적으로 점검한다.

---

## 공통 사전 조건
- 브라우저: Chrome (로그인 상태/권한 허용 여부 기록)
- 네트워크: 정상
- 테스트 계정/프로필: 이름/이메일/전화번호/주소 환경 변수 설정
  - `STEER_PROFILE_NAME`
  - `STEER_PROFILE_EMAIL`
  - `STEER_PROFILE_PHONE`
  - `STEER_PROFILE_ADDRESS`
- 시스템 의존성 확인
  - `cliclick` 설치 여부 (Settings > Missing Dependencies 확인)

---

## 1) 항공권/숙박 (Flights)

### 시나리오 A (왕복 + 예산)
- 입력: "3월 10일 서울에서 도쿄 왕복 30만원 이하 항공권"
- 기대: 출발/도착/날짜 자동 입력 + 검색 결과 요약 + 승인 단계 진입
- 체크
  - [ ] 출발/도착 자동 입력 성공
  - [ ] 날짜 입력 성공
  - [ ] 요약(가격/시간/경유) 추출 성공
  - [ ] 승인 단계 표시

### 시나리오 B (편도 + 시간대)
- 입력: "다음주 월요일 부산에서 제주 편도 오전 출발"
- 체크
  - [ ] 편도 인식
  - [ ] 시간대 조건 반영
  - [ ] 요약 정상

### 시나리오 C (다구간)
- 입력: "서울->도쿄->오사카 3박 4일"
- 체크
  - [ ] 다구간 의도 인식
  - [ ] 플랜 생성 정상
  - [ ] 실패 시 재시도 로그 확보

---

## 2) 가격비교 쇼핑

### 시나리오 A (기본)
- 입력: "아이폰 15 프로 256GB 최저가"
- 체크
  - [ ] 검색어 자동 입력
  - [ ] 검색 제출 성공
  - [ ] 가격 요약 3개 추출

### 시나리오 B (옵션 포함)
- 입력: "에어팟 프로 2세대 화이트"
- 체크
  - [ ] 옵션 파싱
  - [ ] 검색 결과 요약

### 시나리오 C (가격 범위)
- 입력: "무선 청소기 30만원 이하"
- 체크
  - [ ] 가격 조건 반영
  - [ ] 최저가 요약

---

## 3) 웹폼 자동작성

### 시나리오 A (회원가입)
- 입력: "회원가입 폼 자동 작성"
- 체크
  - [ ] 이름/이메일/전화/주소 입력
  - [ ] 민감정보 자동 입력 차단
  - [ ] 승인 전 제출 대기

### 시나리오 B (예약 신청)
- 입력: "예약 신청 폼 채워줘"
- 체크
  - [ ] 필드 매칭 성공
  - [ ] 요약 표시

---

## 4) 승인 정책 검증
- Allow Once: 승인 후 1회만 자동 승인되는지 확인
- Allow Always: 동일 intent+action에 대해 지속 승인되는지 확인
- Deny: 동일 intent+action이 차단되는지 확인

체크
- [ ] Allow Once 동작
- [ ] Allow Always 동작
- [ ] Deny 동작

---

## 5) 실패/재시도 검증
- 검색 버튼 미탐지 → 스크롤 후 재시도
- 자동 입력 실패 → 컨텍스트 로그 확인

체크
- [ ] 1차 실패 후 재시도 발생
- [ ] 실패 컨텍스트 로그 기록

---

## 6) 자동화 신뢰성 (Phase 2)

### 시나리오 A (UI 자동 클릭)
- 입력: "검색 버튼 눌러줘"
- 체크
  - [ ] cliclick 설치 상태 확인
  - [ ] 버튼 클릭 성공/실패 로그 확인

### 시나리오 B (브라우저 자동 입력)
- 입력: "네이버에서 카메라 검색"
- 체크
  - [ ] 입력 실패 시 스크롤 재시도
  - [ ] 페이지 컨텍스트 로그 수집

### 시나리오 C (요약 추출)
- 입력: "최저가 요약해줘"
- 체크
  - [ ] 요약 텍스트 존재
  - [ ] 빈 결과 시 실패 로그 확인
