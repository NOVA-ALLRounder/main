# Rules directory guide

이 디렉터리는 "모델 프롬프트 규칙"과 "정책/필터"를 구분해서 관리하기 위한 공간입니다.

## 역할
- LLM 프롬프트 규칙: 환각 방지, 말투/톤, 응답 형식 등 모델 행동 가이드.
- 정책/필터: 데이터 접근 제약, 민감 경로 제외, 콘텐츠 필터링 등 RAG/파이프라인 정책.

## 사용 지침
- 프롬프트 규칙은 `agent_instructions.md`처럼 LLM 응답 가이드를 정의합니다.
- 정책/필터 정의는 코드/설정(`core/config/`, `core/data_pipeline/policies/`)에 두고, 이 폴더에는 규칙 설명만 남깁니다. (`configs/`는 현재 평가/실험 설정 용도)
- 새 규칙을 추가할 때는 범주를 명시하고(프롬프트 vs 정책) 다른 범주와 섞지 않습니다.
- 코드 변경 시 관련 규칙 문서도 함께 업데이트하고, 문서 없는 규칙은 추가하지 않습니다.

## 하지 말 것
- 정책과 프롬프트 규칙을 한 문서에 혼합하지 않습니다.
- 코드 내 하드코딩된 규칙을 두지 않습니다(문서 + 설정으로 관리).
